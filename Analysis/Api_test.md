# دليل تشغيل السيرفر وحل مشكلة اتصال الـ API

هذا المستند يلخص المشكلة السابقة التي حدثت في الاتصال، ويوضح الخطوات العملية لحلها وتشغيل السيرفر مرة أخرى في حال توقفه مستقبلاً.

---

## 🔍 1. ملخص المشكلة السابقة (تعذر الاتصال بالـ API)

كانت المشكلة ناتجة عن نقطتين رئيسيتين:

1. **تقييد واجهة الاستماع (Localhost Bound):**
   * السيرفر (Laravel) كان يعمل على عنوان المضيف المحلي `127.0.0.1:8000` فقط.
   * **النتيجة:** هذا العنوان لا يسمح بأي اتصالات قادمة من خارج الكمبيوتر نفسه (مثل محاكي الأندرويد Emulator، أو الهاتف المحمول المتصل بالواي فاي)، مما أدى لظهور خطأ فشل الاتصال.

2. **اختلاف عنوان الـ IP في التطبيق:**
   * كان التطبيق مضبوطاً افتراضياً في الكود على الـ IP القديم `192.168.1.2` بينما عنوان جهازك الفعلي حالياً على شبكة الواي فاي هو `192.168.8.107`.

---

## 🛠️ 2. خطوات التشغيل وحل المشكلة مستقبلاً

إذا توقف السيرفر أو لم يعمل الاتصال مرة أخرى، اتبع الخطوات التالية بالترتيب:

### الخطوة الأولى: معرفة الـ IP الحالي لجهازك
قد يتغير عنوان الـ IP الخاص بجهاز الكمبيوتر عند إعادة الاتصال بالشبكة. لمعرفته:
1. افتح نافذة **PowerShell** أو **Command Prompt (CMD)**.
2. اكتب الأمر التالي:
   ```powershell
   ipconfig
   ```
3. ابحث عن **IPv4 Address** تحت كرت الواي فاي (Wireless LAN adapter Wi-Fi).
   * *مثال الحالي:* `192.168.8.107`

---

### الخطوة الثانية: إغلاق أي سيرفر قديم يعمل على نفس البورت (8000)
إذا كان البورت 8000 محجوزاً مسبقاً، لن تتمكن من تشغيل السيرفر بشكل صحيح. لإغلاقه:

* **باستخدام PowerShell (كأدمن):**
  ```powershell
  # لمعرفة المعرف (PID) للعملية التي تستخدم البورت 8000
  Get-NetTCPConnection -LocalPort 8000 | Format-Table LocalAddress, LocalPort, State, OwningProcess

  # لإيقاف العملية (استبدل PID بالرقم المكتوب تحت OwningProcess)
  Stop-Process -Id <PID> -Force
  ```

---

### الخطوة الثالثة: تشغيل سيرفر الـ API بشكل صحيح
اذهب إلى مجلد السيرفر `SASP_Dashbord_API` في التيرمنال وشغله باستخدام المعامل `--host=0.0.0.0` ليقبل الاتصال من الهواتف والمحاكيات:

```bash
cd "d:\All My Project\GitHub_Project\GSP Projects\SASP\SASP App Api\SASP_Dashbord_API"
php artisan serve --host=0.0.0.0 --port=8000
```
> 💡 **ملاحظة:** تحديد `--host=0.0.0.0` يضمن عمل السيرفر على الـ localhost وعلى الـ IP المحلي للشبكة في نفس الوقت.

---

### الخطوة الرابعة: ضبط العنوان في تطبيق Flutter
تأكد من أن التطبيق يوجه الطلبات إلى الـ IP الصحيح للجهاز وبورت السيرفر:

1. **من داخل التطبيق (أسهل طريقة):**
   * في شاشة تسجيل الدخول في التطبيق، اضغط على زر **الترس (Settings)** في الأعلى.
   * قم بتحديث الرابط واكتبه كالتالي:
     `http://192.168.8.107:8000/api` (مع استبدال `192.168.8.107` بالـ IP الحالي إذا تغير).

2. **من داخل الكود (كخيار افتراضي احتياطي):**
   * افتح ملف [sync_service.dart](file:///d:/All%20My%20Project/GitHub_Project/GSP%20Projects/SASP/SASP%20App%20Api/SASP_App_API/lib/core/network/sync_service.dart).
   * تأكد من أن متغير `baseUrl` يحتوي على العنوان الصحيح:
     ```dart
     String baseUrl = 'http://192.168.8.107:8000/api';
     ```
