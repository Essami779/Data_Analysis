# دليل رفع وتهيئة لوحة تحكم لارافل (SASP Dashboard API) على استضافة Hostinger

يتناول هذا الدليل الخطوات التفصيلية لرفع لوحة التحكم المبنية بلارافل إلى استضافة Hostinger، وتهيئة قاعدة البيانات والمجلدات، وربط تطبيق الهاتف الذكي (Flutter) بها ليعمل النظام بشكل متكامل.

---

## 1. تجهيز مشروع لارافل محلياً (Laravel Preparation)

قبل رفع المشروع، يجب تحضيره محلياً للتأكد من خلوه من الملفات غير الضرورية ولضمان سرعة الرفع:

1. **بناء ملفات الواجهة (Vite)**: إذا كانت لوحة التحكم تستخدم ملفات Css/Js مجمعة، قم بتشغيل الأمر التالي في مجلد `SASP_Dashbord_API`:
   ```bash
   npm run build
   ```
   سيقوم هذا الأمر بإنتاج مجلد `build` داخل المجلد `public` والذي يحتوي على الأصول الجاهزة للإنتاج.

2. **ضغط ملفات المشروع**:
   قم بضغط كافة ملفات مجلد `SASP_Dashbord_API` في ملف ZIP واحد، مع **تجنب** ضغط المجلدات التالية لتقليل الحجم وتسريع الرفع (حيث سيتم توليدها أو رفعها بطرق أخرى):
   - `node_modules` (غير مطلوب نهائياً في الاستضافة المشتركة).
   - `vendor` (يفضل ضغطه إذا كنت لن تشغل `composer install` عبر الـ SSH، أو تركه وتشغيل التثبيت في الاستضافة).
   - `.git` و `.idea` و `.vscode` (ملفات تطوير داخلية).
   - `storage/logs/*` (تأكد من إبقاء هيكل مجلد storage فارغاً من سجلات الأخطاء القديمة).

---

## 2. إعداد قاعدة البيانات على Hostinger (Database Setup)

1. سجل الدخول إلى لوحة تحكم **Hostinger hPanel**.
2. انتقل إلى قسم **Databases (قواعد البيانات)** -> **MySQL Databases**.
3. قم بإنشاء قاعدة بيانات جديدة:
   - **اسم قاعدة البيانات (Database Name)**: مثلاً `u123456789_sasp_db`.
   - **اسم المستخدم (Database User)**: مثلاً `u123456789_sasp_user`.
   - **كلمة المرور (Password)**: اختر كلمة مرور قوية جداً واحفظها.
4. اضغط على **Create**.
5. بعد إنشاء قاعدة البيانات، اضغط على زر **Enter phpMyAdmin** بجانب قاعدة البيانات المنشأة.
6. محلياً، قم بتصدير قاعدة البيانات الحالية (ملف `.sql`) من phpMyAdmin المحلي.
7. في phpMyAdmin الخاص بـ Hostinger، اختر **Import (استيراد)**، وقم برفع ملف `.sql` الذي قمت بتصديره ليتم بناء الجداول والبيانات.

---

## 3. رفع ملفات المشروع وضبط المسارات (File Upload & Structure)

هناك طريقتان لرفع المشروع، وتعتبر **الطريقة الأولى (الآمنة)** هي الأفضل لضمان حماية ملفات لارافل الحساسة (مثل ملف `.env`).

### الطريقة الأولى: الهيكلية الآمنة (فصل المجلد العام عن ملفات النظام) - موصى بها

نفترض أن مسار مجلدك الرئيسي في الاستضافة هو: `/home/u123456789/`
ومجلد الدومين الخاص بك هو: `/home/u123456789/public_html/`

1. ارفع ملف الـ ZIP الخاص بالمشروع إلى المسار الرئيسي للمستخدم `/home/u123456789/` (أي خارج `public_html`).
2. قم بفك الضغط (Extract) هناك في مجلد جديد، لنسمه مثلاً `sasp_api`.
3. الآن، ادخل إلى مجلد `sasp_api` وانقل **محتويات مجلد `public` فقط** إلى المجلد الرئيسي للموقع `public_html`.
4. بعد نقل محتويات المجلد `public` إلى `public_html` (بما فيها ملفات `index.php` و `.htaccess` ومجلد `build` و `assets`)، افتح ملف `public_html/index.php` للتعديل واضبط المسارات لتبدو كالتالي لكي يتعرف على ملفات النواة والـ vendor:
   * ابحث عن السطر الذي يستدعي `autoload.php` (عادة حوالي السطر 34) وقم بتعديله ليصبح:
     ```php
     require __DIR__.'/../sasp_api/vendor/autoload.php';
     ```
   * ابحث عن السطر الذي يستدعي `app.php` (عادة حوالي السطر 47) وقم بتعديله ليصبح:
     ```php
     $app = require_once __DIR__.'/../sasp_api/bootstrap/app.php';
     ```
5. احفظ الملف.

---

### الطريقة الثانية: الرفع المباشر داخل `public_html` (التقليدية)

إذا أردت بقاء المشروع بأكمله داخل مجلد واحد أو كنت تستخدم مجلداً فرعياً (مثلاً `public_html/api/`):
1. ارفع ملف الـ ZIP الخاص بالمشروع إلى مجلد `public_html` وقم بفك ضغطه هناك.
2. للتأكد من حماية ملفات `.env` وإعادة توجيه الزوار إلى مجلد `public` تلقائياً، قم بإنشاء أو تعديل ملف `.htaccess` داخل `public_html` ووضع الأكواد التالية فيه:
   ```apache
   <IfModule mod_rewrite.c>
       RewriteEngine On
       # توجيه كافة الطلبات إلى مجلد public
       RewriteRule ^(.*)$ public/$1 [L]
   </IfModule>
   
   # حظر الوصول إلى ملف .env والملفات الحساسة الأخرى
   <Files .env>
       Order allow,deny
       Deny from all
   </Files>
   ```

## 4. تهيئة ملف الإعدادات البيئية (`.env`)

افتح ملف `.env` الموجود في مجلد المشروع الرئيسي (مثلاً `sasp_api/.env` أو `public_html/.env`) وقم بتعديل القيم التالية لتتناسب مع الإنتاج واستضافة Hostinger:

```ini
# تحويل التطبيق لوضع الإنتاج وإيقاف تشغيل وضع تصحيح الأخطاء (Debug) للأمان
APP_ENV=production
APP_DEBUG=false

# تحديث رابط التطبيق إلى النطاق الفعلي الخاص بك (بدون شرطة مائلة في النهاية)
APP_URL=https://sasp.astrodevye.com

# إعدادات قاعدة البيانات الجديدة التي قمت بإنشائها على Hostinger
DB_CONNECTION=mysql
DB_HOST=127.0.0.1          # غالباً ما يكون localhost أو 127.0.0.1 في Hostinger
DB_PORT=3306
DB_DATABASE=u123456789_sasp_db
DB_USERNAME=u123456789_sasp_user
DB_PASSWORD=your_strong_password

# إعدادات الجلسات والتخزين المؤقت
SESSION_DRIVER=database
CACHE_STORE=database

# إعدادات البريد الإلكتروني (SMTP)
# تأكد من استخدام SMTP الخاص بـ Hostinger أو Gmail وتفعيل المنفذ الآمن
MAIL_MAILER=smtp
MAIL_HOST=smtp.hostinger.com # أو smtp.gmail.com
MAIL_PORT=465
MAIL_USERNAME=your-email@sasp.astrodevye.com
MAIL_PASSWORD=your_email_password
MAIL_ENCRYPTION=ssl
MAIL_FROM_ADDRESS="your-email@sasp.astrodevye.com"
MAIL_FROM_NAME="${APP_NAME}"
```

## 5. صلاحيات المجلدات وتنظيف ذاكرة التخزين المؤقت (Permissions & Cache)

لكي يعمل لارافل دون مشاكل متعلقة بالكتابة والقراءة على الخادم، يجب تطبيق التالي:

1. **صلاحيات المجلدات**:
   تأكد من أن المجلدات التالية تملك صلاحية كتابة وقراءة صحيحة (عادة `775` أو `755` اعتماداً على إعدادات خادم Hostinger):
   - `storage` (وكافة المجلدات الفرعية داخله مثل `storage/framework` و `storage/logs`).
   - `bootstrap/cache`.

2. **ربط مجلد التخزين (Storage Link)**:
   يتطلب لارافل إنشاء رابط رمزي (Symlink) لكي تصبح الصور والملفات المرفوعة متاحة للعامة.
   بما أن Hostinger لا يتيح تشغيل أمر `php artisan storage:link` مباشرة بدون SSH في الباقات العادية، يمكنك القيام بذلك بإحدى الطرق التالية:
   * **عن طريق خادم SSH (إذا كانت باقتك تدعم ذلك)**: اتصل بالـ SSH للموقع وشغل الأمر داخل مجلد المشروع:
     ```bash
     php artisan storage:link
     ```
   * **عن طريق ملف الراوتس (web.php)**: قم بإضافة الراوت المؤقت التالي في ملف routes/web.php محلياً قبل رفعه أو مباشرة بالخادم، ثم قم بزيارة الرابط `https://sasp.astrodevye.com/run-link` مرة واحدة:
     ```php
     Route::get('/run-link', function () {
         $targetFolder = base_path('storage/app/public');
         $linkFolder = $_SERVER['DOCUMENT_ROOT'] . '/storage';
         if (!file_exists($linkFolder)) {
             symlink($targetFolder, $linkFolder);
             return 'تم إنشاء رابط التخزين بنجاح!';
         }
         return 'رابط التخزين موجود بالفعل!';
     });
     ```
     *(بعد زيارة الرابط ونجاح العملية، يفضل مسح هذا الكود للأمان)*.

3. **تنظيف الكاش (Clear Cache)**:
   لتجنب تضارب التكوينات القديمة، قم بزيارة الروابط التالية عبر إضافة كود مؤقت أو عبر SSH:
   ```bash
   php artisan config:clear
   ```
   أو عبر الكود:
   ```php
   Route::get('/clear-all-cache', function() {
       Artisan::call('config:clear');
       Artisan::call('cache:clear');
       Artisan::call('route:clear');
       Artisan::call('view:clear');
       return "تم تنظيف كافة أنواع الكاش بنجاح!";
   });
   ```

## 6. ربط تطبيق فلوتر (Flutter Mobile App Integration)

الآن بعد رفع لوحة التحكم بنجاح، يحتاج تطبيق الهاتف إلى توجيه طلباته إلى السيرفر الجديد بدلاً من السيرفر المحلي (`192.168.x.x`).

1. افتح مشروع فلوتر `SASP_App_API`.
2. انتقل إلى ملف السيرفر والاتصال بالشبكة:
   `lib/core/network/sync_service.dart`
3. قم بتحديث المتغير `baseUrl` الافتراضي في السطر 11:
   ```dart
   String baseUrl = 'https://sasp.astrodevye.com/api/sasp';
   ```
   *(قم باستبدال `sasp.astrodevye.com` بالنطاق الفعلي للموقع الذي قمت برفع اللوحة عليه).*

4. **ملاحظة هامة حول قاعدة البيانات المحلية (SQLite)**:
   يقوم التطبيق بجلب الـ `baseUrl` المخزن مسبقاً في جدول الـ `settings` بقاعدة بيانات التطبيق باسم مفتاح `api_base_url`.
   لضمان تفعيل الرابط الجديد فوراً لجميع المستخدمين الذين يملكون التطبيق حالياً بدون مشاكل الكاش القديم:
   - يفضل تعديل كود المزامنة التلقائي في دالة `_loadBaseUrl` ليقوم بعمل تحديث للمسار القديم تلقائياً ليتحول إلى النطاق الجديد كما هو معمول به مسبقاً في الكود.
   - أو يمكنك ببساطة تحديث الكود في التطبيق ليقوم بمسح الكاش المحلي مرة واحدة أو مسح قاعدة البيانات SQLite عند الترقية (Migration).

## 7. التحقق والتشغيل الميداني (Testing)

بمجرد إتمام الخطوات أعلاه:
1. قم بفتح لوحة التحكم من المتصفح عبر الرابط `https://sasp.astrodevye.com` وتأكد من ظهور واجهة الدخول بشكل صحيح.
2. قم بتسجيل الدخول بأحد الحسابات للتحقق من الاتصال بقاعدة البيانات بشكل سليم.
3. قم بتشغيل تطبيق Flutter والاتصال بالسيرفر، وتأكد من نجاح عمليتي تسجيل الدخول والمزامنة (Sync) بنجاح وسحب البيانات من السيرفر الجديد.
